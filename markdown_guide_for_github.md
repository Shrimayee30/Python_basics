# Markdown Guide for a GitHub Homepage

Markdown is a simple way to format text on GitHub. Students can use it to build a clean and friendly profile `README.md`.

## 1. Headings

Use `#` symbols to make titles and section headings.

```md
# My GitHub Page
## About Me
### Projects
```

## 2. Bold and Italic Text

Use `**` for bold and `*` for italic.

```md
I am **learning Python**.
I like *building small projects*.
```

## 3. Lists

Use `-` for bullet points.

```md
- Python
- Scratch
- Web design
```

Use numbers for ordered lists.

```md
1. Learn Markdown
2. Make a README
3. Share projects
```

## 4. Links

Use brackets for the text and parentheses for the URL.

```md
[Visit GitHub](https://github.com)
```

## 5. Images

Use `!` before a link to show an image.

```md
![My photo](https://example.com/photo.png)
```

## 6. Code

Use backticks for short code.

```md
Use the `print()` function in Python.
```

Use triple backticks for a code block.

```md
```python
print("Hello, GitHub!")
```
```

## 7. Blockquotes

Use `>` to show a quote or a highlighted note.

```md
> I am excited to keep learning and building.
```

## 8. Line Breaks and Paragraphs

Leave a blank line between paragraphs.

```md
Hello, my name is Maya.

I am a student who enjoys coding.
```

## 9. Simple Table

Tables can organize skills or project ideas.

```md
| Skill | Level |
|-------|-------|
| Python | Beginner |
| HTML | Beginner |
```

## 10. Example GitHub Homepage

Students can start with something like this:

```md
# Hi, I'm Maya

## About Me
- I am learning Python and GitHub.
- I enjoy solving problems with code.
- I want to build fun beginner projects.

## My Skills
- Python
- Markdown
- GitHub

## My Goal
I want to use this page to share my work and track my progress.
```

## Tips

- Keep it simple at first.
- Use short sections and clear headings.
- Add projects later as students complete them.
- Preview the file on GitHub to check formatting.
